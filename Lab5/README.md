##  Завдання (варіант №5)

1️ Що таке Generics? Які їхні переваги?
Generics (узагальнені типи) — це механізм C#, який дозволяє створювати класи, методи та інтерфейси, що працюють із будь-яким типом даних без втрати типобезпеки.
Переваги:

Відсутність потреби в приведенні типів (casting)

Типобезпека під час компіляції

Підвищена продуктивність

Гнучкість і повторне використання коду

2️ Які основні відмінності між ArrayList і List<T>?
Характеристика	ArrayList	List<T>
Тип даних	object (усі типи)	Узагальнений T
Безпека типів	Немає	Є
Продуктивність	Нижча (boxing/unboxing)	Вища
Рекомендація	Застарілий	Рекомендується

3️ Чим відрізняється Dictionary<TKey, TValue> від List<T>?
Dictionary — колекція пар ключ–значення, кожен ключ унікальний.

List — звичайна послідовна колекція елементів, доступ за індексом.

У Dictionary пошук швидший (O(1)), у List — повільніший (O(n)).

4️ У чому перевага LINQ над класичними циклами?
Лаконічний, декларативний стиль запитів.

Легке фільтрування, сортування, групування.

Може працювати з масивами, списками, XML, базами даних.

Менше коду — менше шансів на помилки.

5️ Як працює ключове слово finally?
Виконується завжди, незалежно від того, чи виник виняток.

Застосовується для звільнення ресурсів (файли, з’єднання тощо).

csharp
Копіювати код
try {
    // код
}
catch (Exception ex) {
    Console.WriteLine(ex.Message);
}
finally {
    Console.WriteLine("Виконується завжди");
}
6️ Коли доцільно створювати власні класи винятків?
Коли стандартні винятки не відображають логіку програми.

Щоб зробити повідомлення про помилки зрозумілішими.

Для доменних перевірок (наприклад, InvalidRatingException, InsufficientFundsException).
