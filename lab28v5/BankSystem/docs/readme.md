#  Bank Management System (Mini-Project)

Професійна реалізація банківської системи на платформі **.NET 8**, виконана згідно з вимогами Варіанту №5. Проєкт демонструє навички побудови чистої архітектури, використання патернів проєктування та дотримання принципів SOLID.

---

##  Технологічний стек
* **Мова:** C# 12 / .NET 8
* **Архітектура:** Layered Architecture (Шарова архітектура)
* **Формат даних:** JSON (System.Text.Json)
* **Тестування:** xUnit

---

##  Архітектура та SOLID
Проєкт розділений на чотири логічні шари:
1.  **Domain:** Сутністі предметної області (`Account`, `Customer`, `Transaction`) та контракти (інтерфейси).
2.  **Infrastructure:** Реалізація збереження даних (`JsonRepository`).
3.  **Application:** Бізнес-логіка системи (`BankService`).
4.  **Console:** Інтерфейс користувача та точка входу.

### Дотримання SOLID:
* **SRP (Single Responsibility):** Кожен клас відповідає за свою задачу (напр. `JsonRepository` тільки за збереження).
* **LSP (Liskov Substitution):** `SavingsAccount` може використовуватись замість базового `Account` без порушення логіки.
* **DIP (Dependency Inversion):** Сервіси залежать від абстракцій (`IRepository`), а не від конкретних реалізацій.

### Патерни проєктування:
* **Repository Pattern:** Уніфікований доступ до даних незалежно від типу сховища.
* **Dependency Injection:** Впровадження залежностей через конструктори для полегшення тестування.

---

##  Предметна область (Класів: 8)
Система містить наступні ключові сутності:
* `Account` (Базовий рахунок)
* `SavingsAccount` (Ощадний рахунок з відсотками)
* `Customer` (Клієнт банку)
* `Transaction` (Логівання операцій)
* `BankBranch` (Відділення)
* `BankCard` (Прив'язка карти)
* `IRepository` (Інтерфейс доступу)
* `BankService` (Серце бізнес-логіки)

---

##  Функціональні можливості (Use Cases)
1.  **Створення рахунків:** Автоматична реєстрація нових клієнтів.
2.  **Валідація транзакцій:** Перевірка балансу та обробка виняткових ситуацій (Scenario Failure).
3.  **Persistence:** Автоматичне збереження всього стану системи у файл `accounts.json`.
4.  **LINQ Analytics:** Пошук та фільтрація рахунків за складними критеріями (напр. `GetRichAccounts`).

##  Тестування
Проєкт покритий тестами, що включають:
* **Unit Tests:** Тестування логіки нарахувань та зняття коштів.
* **Negative Testing:** Перевірка реакції системи на некоректні дані (недостатньо коштів).
* **Integration Tests:** Перевірка взаємодії сервісу з файловою системою.
